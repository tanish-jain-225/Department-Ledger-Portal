# Project Demo Script Plan

Duration: 2-3 minutes

Goal: deliver a winning, high-clarity walkthrough of the complete project flow with visible proof.

Last validated against current build: April 11, 2026

## Winning Structure

1. Hook the problem quickly.
2. Show AI data capture in action.
3. Show AI outcome intelligence in action.
4. Prove governance and institutional readiness.
5. Close with deployment and impact.

## Full Recording Script (Verbatim)

### 0:00-0:10 | Problem Hook

Screen action:
- Open the landing page hero section.

Say:
- "Academic departments still lose time and data quality with fragmented spreadsheets and manual entry. Department Ledger Portal solves this with one secure, AI-assisted academic ledger."

### 0:10-0:25 | Product and Roles

Screen action:
- Keep landing section and feature cards visible.

Say:
- "This platform is role-based for students, faculty, and admins. Students maintain academic and career records, faculty can review and search quickly, and admins enforce governance with approvals, audit tracking, and secure workflows."

### 0:25-0:35 | Sign-in Context

Screen action:
- Click Sign In and show authenticated navigation state.

Say:
- "From sign-in onward, route access is role-aware, so users only see what they are authorized to use."

### 0:35-1:10 | Student Flow: Smart Analysis

Screen action:
- Open student profile.
- Go to a ledger section with Smart Analysis.
- Upload one sample document.

Say:
- "Here is Smart Analysis. Instead of manually typing entries, the student uploads a supporting file and Gemini extracts structured values for the selected section."
- "The extracted data is mapped into the form, so users can validate and save quickly with fewer errors."

Say while waiting for response:
- "This flow is powered by the autofill endpoint and includes validation for file type and payload integrity before AI processing."

### 1:10-1:25 | Save and Confirm Data Capture

Screen action:
- Click Add/Save and show newly added record in list/timeline.

Say:
- "Now the extracted information is committed to the ledger and becomes part of the student history for later review and analytics."

### 1:25-1:55 | Student Flow: Career Pulse

Screen action:
- Trigger AI readiness analysis.
- Open generated report.
- Point at score, summary, strengths, gaps, recommendations, and roadmap.

Say:
- "Next is Career Pulse, which converts student profile and record data into actionable readiness intelligence."
- "The report includes a score, executive summary, strengths, critical gaps, and a practical action roadmap."
- "This gives students and mentors a structured plan instead of generic feedback."

### 1:55-2:20 | Faculty and Admin Governance

Screen action:
- Switch to faculty/admin pages.
- Show student search/filter.
- Show requests and audit view.

Say:
- "Faculty can discover students quickly using search and filters, and review complete records in one place."
- "Admins handle role and deletion requests with auditable actions, giving departments governance and accountability built into daily workflows."

### 2:20-2:35 | Deployment and Impact Close

Screen action:
- Show live app URL and repository link.

Say:
- "The solution is live on Vercel, integrated with Firebase and Gemini, and validated with lint, tests, production build, and end-to-end checks."
- "Department Ledger Portal makes academic data faster to capture, safer to govern, and more useful for placement outcomes."

## Backup Branch (If AI or Network Is Slow)

If Smart Analysis or Career Pulse takes too long, say this and continue:

- "AI processing is in progress. In parallel, here is a previously generated report from the Intelligence Vault showing the exact output format and decision guidance."
- "The platform includes retry handling and timeout-safe messaging so user flow remains stable."

Then jump directly to Faculty/Admin governance section.

## On-Screen Sequence Checklist

1. Landing hero and value proposition visible.
2. Sign-in and role-based context shown.
3. Smart Analysis upload demonstrated.
4. Autofilled fields visible.
5. Record saved and visible in ledger list.
6. Career Pulse report generated or opened from vault.
7. Faculty search/filter shown.
8. Admin requests and audit page shown.
9. Live URL and repo shown at close.

## Delivery Tips for Higher Score

1. Keep cursor movement deliberate and minimal.
2. Never wait silently during AI calls; narrate safeguards and flow.
3. Use one concrete student example from upload to saved record.
4. Use "AI-assisted" wording, not absolute automation claims.
5. End with measurable value: speed, governance, and decision quality.

## Pre-Recording Setup

1. Keep one student account and one admin account pre-logged or ready.
2. Keep one sample marksheet/certificate file ready.
3. Keep one previously generated readiness report available as fallback.
4. Confirm notifications and audit entries are present for visual proof.
5. Record at readable zoom and consistent resolution.
